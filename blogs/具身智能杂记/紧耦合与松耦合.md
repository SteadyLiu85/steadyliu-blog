## 一、 松耦合 与 紧耦合

### 1. 松耦合（Loosely Coupled）

在松耦合系统中通常将LiDAR的前端里程计算法视为黑盒。系统只关心其输出的位姿（Pose），而不干涉其内部的点云匹配过程。

**① 独立解算**：LiDAR模块独立运行（如基于ICP或NDT），计算出当前帧的旋转 $\mathbf{R}$ 和平移 $\mathbf{p}$；IMU模块独立进行积分，提供高频的相对位姿推算。

**② 高层融合**：在后端的融合层（如标准的KF或简单的位姿图优化中），将LiDAR输出的位姿作为观测值，去更新IMU推算出的先验状态。

**优劣：**

- **优势**：利于解耦与模块化。LiDAR前端与IMU后端可以并行开发，且非常容易替换（比如将LIO替换为VIO）。
- **劣势**：信息利用率低下。当系统处于特征退化场景时，LiDAR前端在点云匹配阶段就会失败，根本无法输出有效的位姿给融合层，导致整个系统迅速发散。

### 2. 紧耦合（Tightly Coupled）

紧耦合直接在原始点云层面进行整合。

**① 联合处理**：不再存在独立的LiDAR里程计，LiDAR的原始扫描点（Raw Points）和IMU的测量数据直接被送入同一个迭代卡尔曼滤波器（IEKF）或因子图中。

**② 互补校正**：利用IMU的高频积分推算出当前时刻的精确预测位姿，利用该位姿对点云进行**运动畸变去除**，随后直接计算点云特征到局部地图的残差（Residual），并通过残差联合更新IMU的零偏（Bias）和系统的运动状态。

**优劣：**

- **优势**：极高的精度与鲁棒性。即使在点云特征稀疏的场景下，虽然约束不足以求解完整的6自由度位姿，但紧耦合框架依然能利用这部分点云在法线方向上的约束来修正IMU在该方向的漂移，最大化了数据的价值。
- **劣势**：数学推导复杂，矩阵维度极高，对实时计算和矩阵库（如Eigen）的数值稳定性处理提出了极高要求。

## 二、 松耦合的架构

基于上述原理，松耦合 LIO 的本质是用ESKF（误差状态卡尔曼滤波）来处理一个纯粹的 $\mathbf{R}, \mathbf{p}$ 观测问题。在代码架构上通常分为三个线程或模块：

**① 时间戳同步**：硬件或软件层面将高频IMU数据与低频雷达帧对齐。

**② IMU 预处理与去畸变**：利用ESKF中IMU的预测位姿，对一帧（通常为100ms）内不断旋转平移的激光雷达产生的点云进行运动补偿（Deskewing）。

**③ 状态更新**：从ESKF获取预测的先验位姿，交给LiDAR里程计算法作为配准的初值（Initial Guess）。等ICP配准结束后，将其输出的后验位姿作为观测变量 $\mathbf{z}$，放回ESKF中计算卡尔曼增益并更新状态。

## 三、 紧耦合数学推导

目前紧耦合方案往往基于**迭代误差状态卡尔曼滤波（IEKF）**，这也是 FAST-LIO 系列的核心理论。

### 1. 预测步

**① 状态变量**

我们将系统的真实状态定义为 $\mathbf{x}$，并在流形（Manifold）空间上定义误差状态 $\delta\mathbf{x}$：

$$\mathbf{x} = [\mathbf{R}, \mathbf{p}, \mathbf{v}, \mathbf{b}_g, \mathbf{b}_a]^T$$

$$\delta\mathbf{x} = [\delta\boldsymbol{\theta}, \delta\mathbf{p}, \delta\mathbf{v}, \delta\mathbf{b}_g, \delta\mathbf{b}_a]^T$$

**② 连续时间运动方程**

根据IMU的物理特性，构建系统的连续时间运动学方程（考虑重力 $\mathbf{g}$ 和噪声 $\mathbf{n}$）：

$$\dot{\mathbf{p}} = \mathbf{v}$$

$$\dot{\mathbf{v}} = \mathbf{R}(\mathbf{a}_m - \mathbf{b}_a - \mathbf{n}_a) + \mathbf{g}$$

$$\dot{\mathbf{R}} = \mathbf{R} \lfloor (\boldsymbol{\omega}_m - \mathbf{b}_g - \mathbf{n}_g)_{\times} \rfloor$$

**③ 协方差**

通过离散化，利用状态转移矩阵 $\mathbf{F}$ 和噪声协方差矩阵 $\mathbf{Q}$ 向前推算误差状态的协方差：

$$\mathbf{P}_{k+1|k} = \mathbf{F} \mathbf{P}_{k|k} \mathbf{F}^T + \mathbf{Q}$$

### 2. 更新步

紧耦合的观测方程直接构建在点云残差上。由于非线性强，我们需要使用**迭代卡尔曼滤波（IEKF）**来逼近最优解。

**① 残差方程**

对于第 $j$ 个特征点，其观测残差 $\mathbf{r}_j$ 通常表现为该点投影到局部地图（点到面 / 点到线）的距离：

$$\mathbf{r}_j = h_j(\hat{\mathbf{x}}_k) - \mathbf{z}_j$$

其中 $h_j(\cdot)$ 是高度非线性的观测函数。

**② 雅可比矩阵**

对残差函数在当前的线性化点 $\hat{\mathbf{x}}_k$ 处关于误差状态 $\delta\mathbf{x}$ 求导：

$$\mathbf{H}_k = \left. \frac{\partial h(\mathbf{x})}{\partial \delta\mathbf{x}} \right|_{\mathbf{x} = \hat{\mathbf{x}}_k}$$

**③ 迭代计算卡尔曼增益与状态更新**

这一步是平衡“IMU先验信任度”与“LiDAR观测信任度”的核心。标准公式为：

$$\mathbf{K} = \mathbf{P} \mathbf{H}_k^T (\mathbf{H}_k \mathbf{P} \mathbf{H}_k^T + \mathbf{R_v})^{-1}$$

$$\Delta \mathbf{x} = \mathbf{K} \mathbf{r}_k$$

状态更新（引入流形上的加法 $\boxplus$ 保证旋转矩阵的正交性）：

$$\hat{\mathbf{x}}_{k+1} = \hat{\mathbf{x}}_k \boxplus \Delta \mathbf{x}$$

**收敛与协方差更新：**

不断重复计算 $\mathbf{H}_k$, $\mathbf{K}$, $\Delta \mathbf{x}$，直到状态增量 $\Delta \mathbf{x}$ 的范数小于设定的阈值（即收敛）。一旦收敛，执行协方差矩阵的更新：

$$\mathbf{P} = (\mathbf{I} - \mathbf{K} \mathbf{H}) \mathbf{P}$$

这说明，在迭代过程中修改的只是状态的线性化起始点，直到最后一步才确认整个系统的不确定度收敛。

### 3. SMW 恒等式

在紧耦合 LIO 中，如果不做特殊处理，上述逻辑在现实中是不可用的。

**原因在于维度**：如果当前帧提取了 2000 个特征点，那么观测矩阵 $\mathbf{H}$ 的维度就是 $2000 \times 18$。在计算卡尔曼增益 $\mathbf{K}$ 时，括号内的求逆部分 $(\mathbf{H}_k \mathbf{P} \mathbf{H}_k^T + \mathbf{R_v})$ 将是一个 $2000 \times 2000$ 的巨大矩阵，实时求逆会死。

在FAST_LIO2中，用**Sherman-Morrison-Woodbury (SMW)** 恒等式进行了等效重构：

**优化前：**

$$\mathbf{K} = \mathbf{P} \mathbf{H}^T (\mathbf{H} \mathbf{P} \mathbf{H}^T + \mathbf{R_v})^{-1}$$

*(矩阵求逆维度： $M \times M$，M为特征点数量)*

**优化后：**

$$\mathbf{K} = (\mathbf{H}^T \mathbf{R_v}^{-1} \mathbf{H} + \mathbf{P}^{-1})^{-1} \mathbf{H}^T \mathbf{R_v}^{-1}$$

*(矩阵求逆维度：状态向量的维度 $\times$ 状态向量的维度，即 $18 \times 18$)*
